# GoodRoad

**Авторы:** `Городкова Ксения`, `Грудцына Виктория`, `Толстограева Виктория`

## О проекте

**GoodRoad** — мобильное приложение для построения пеших инклюзивных маршрутов с учетом состояния дороги, препятствий на пути и персональных ограничений пользователя.

В отличие от обычной навигации по кратчайшему пути, GoodRoad учитывает наличие лестниц, ям, бордюров и других барьеров. Это позволяет строить более подходящие маршруты для пользователей с разными ограничениями и заранее показывать проблемные участки пути. Приложение также поддерживает сбор пользовательских отзывов о доступности дорог и объектов в игровом формате, чтобы данные о городской среде можно было постепенно актуализировать и расширять.

## MVP

Сейчас проект находится на этапе MVP. В текущей версии реализован основной сценарий работы системы: пользователь настраивает ограничения, строит маршрут с учетом препятствий и оставляет отзывы, а модератор проверяет пользовательский контент. Серверная часть покрывает базовую бизнес-логику: аутентификацию, работу с профилем, политику препятствий, отзывы и модерацию.

### Возможности MVP

**Для пользователя**
- Регистрация, вход и восстановление пароля;
- Просмотр и редактирование профиля;
- Выбор препятствий, которых нужно избегать, и их тяжести;
- Построение маршрута с учетом ограничений;
- Создание, редактирование и удаление отзывов;
- Добавление фотографий к отзыву;
- Просмотр статуса модерации и начисленных баллов.

**Для модератора**
- Просмотр отзывов, отправленных на проверку;
- Одобрение и отклонение отзывов.

**Для главного модератора**
- Все возможности модератора;
- Добавление и удаление других модераторов. 

## Что планируем сделать к защите

Дальнейшее направление развития — расширение маршрутизации и социальных сценариев. В частности, построение нескольких вариантов пути, создание волонтерского взаимодействия, расширение системы баллов при помощи геймификации и дальнейшее развитие модераторских и пользовательских сценариев.

## Архитектура

**GoodRoad состоит из двух частей:**
- [Android-клиент](https://github.com/GoodRoad-Project/GoodRoad-Client) отвечает за пользовательский интерфейс, экранные сценарии, работу с картой, маршрутами, профилем и отзывами;
- [Сервер](https://github.com/GoodRoad-Project/GoodRoad-Server) отвечает за REST API, бизнес-логику, аутентификацию, работу с базой данных, препятствиями, отзывами и модерацией.

<b>Общая схема взаимодействия:</b><br>
Android Client → REST API → Service Layer → Repository Layer → PostgreSQL / PostGIS
